Day 3 ‚Äì VPC & Networking

What is a VPC?

VPC (Virtual Private Cloud) is your isolated private network inside AWS ‚Äî like your own mini data center in the cloud. You define:

  • IP address range (CIDR block)
  • Subnets (public/private)
  • Route tables
  • Internet gateways and NAT gateways

Each AWS account comes with a default VPC, but custom VPCs offer better control and security.

Key Components

Subnets

  • Divide your VPC into logical sections (e.g., 10.0.0.0/16 ‚Üí 10.0.1.0/24)
  • Public subnet: Route to Internet Gateway.
  • Private subnet: No direct route to Internet; use NAT Gateway for outbound access.

Route Tables

  • Control where traffic goes within your VPC.
  • Each subnet must be associated with a route table.
  • Examples:
    • 0.0.0.0/0 ‚Üí Internet Gateway (public)
    • 0.0.0.0/0 ‚Üí NAT Gateway (private)

Internet Gateway (IGW)

  • Enables internet access for public subnets.
  • Attach IGW to your VPC and add it to route tables.

NAT Gateway

  • Allows private subnet resources to access the internet securely.
  • Managed by AWS; preferred over NAT Instances.

Security Groups vs NACLs

  • Security Groups: Instance-level, stateful.
  • NACLs: Subnet-level, stateless.
  • Always prefer SGs for fine-grained control.

Peering, Endpoints & Transit Gateway

VPC Peering

  • Private connection between two VPCs (no transitive routing).

VPC Endpoints

  • Access AWS services without Internet Gateway.
  • Interface Endpoint: Most services.
  • Gateway Endpoint: S3 and DynamoDB.

Transit Gateway

  • Central hub for connecting multiple VPCs and on-premises networks.
  • Scalable and simplifies architecture.

CIDR, IPs & DNS

  • Example: 10.0.0.0/16 ‚Üí 65,536 IPs.
  • Each subnet reserves 5 IPs for AWS internal use.
  • Elastic IPs = static public IPs.
  • Route 53 integrates with VPC for internal DNS.

Common Architectures

  • 2-tier: Web (public) + DB (private)
  • 3-tier: Web ‚Üí App ‚Üí DB (multi-subnet)
  • HA Setup: Deploy across multiple AZs.

Exam Tips

  • Default VPC has public subnets and internet access.
  • Security Group = Stateful | NACL = Stateless.
  • Use NAT Gateway for private outbound traffic.
  • No transitive routing in VPC Peering.
  • Gateway Endpoint = private S3/DynamoDB access.
  • One route table per subnet.
  • Non-overlapping CIDR blocks for peering.
