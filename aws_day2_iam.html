Day 2 – IAM, Users, Roles

IAM Overview

  • AWS Identity and Access Management (IAM) controls access to AWS services.
  • Supports users, groups, roles, and policies.
  • Use **least privilege principle** — grant only necessary permissions.
  • IAM is a **global service**, no region selection needed.

Users & Groups

  • Create IAM users for individuals; avoid sharing root credentials.
  • Groups help manage permissions collectively.
  • Assign policies to users/groups for controlled access.

Roles & Policies

  • Roles allow services or users to assume permissions temporarily.
  • Policies define permissions (JSON-based).
  • Use managed policies for simplicity or inline policies for fine-grained control.

Exam Tips

  • Remember: IAM is **global**, not regional.
  • Best practice: use **MFA** for critical accounts.
  • Cross-account access usually involves **roles**, not sharing credentials.
